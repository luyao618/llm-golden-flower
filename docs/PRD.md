# Golden Flower Poker AI - 需求文档 (PRD)

## 1. 项目概述

**项目名称**: Golden Flower Poker AI (炸金花 AI 对战游戏)

**一句话描述**: 一款 Web 端炸金花游戏，玩家与多个由不同 AI 大模型驱动的智能对手对战，每个 AI 对手拥有独立的性格和决策逻辑，并在游戏过程中记录完整的"心路历程"供玩家赛后查阅。

**核心卖点**:
- 每个 AI 对手由不同的大模型 API 驱动（OpenAI / Anthropic / Gemini 等）
- AI 对手拥有独立人格，决策风格各异
- **牌桌聊天**: AI 会在牌桌上说话施压、虚张声势、互相试探，玩家也可以参与对话
- **经验学习**: AI 会回顾之前几局的心路历程，吸取教训调整策略，越打越"聪明"
- AI 在每次决策时记录结构化思考过程 + 第一人称叙事
- 每局结束和游戏结束后可查看每个 AI 的完整心路历程报告

## 2. 用户角色

| 角色 | 描述 |
|------|------|
| 玩家 | 唯一的人类用户，通过浏览器参与游戏 |
| AI 对手 | 1-5 个 AI Agent，由不同 AI Model 驱动，各自独立决策 |

## 3. 炸金花经典规则

### 3.1 基本设置

- **牌组**: 一副标准 52 张扑克牌（去除大小王）
- **人数**: 2-6 人（1 个玩家 + 1-5 个 AI）
- **每人发牌**: 3 张暗牌

### 3.2 牌型大小（从大到小）

| 排名 | 牌型 | 说明 | 示例 |
|------|------|------|------|
| 1 | 豹子 (Three of a Kind) | 三张点数相同 | AAA |
| 2 | 同花顺 (Straight Flush) | 三张花色相同且点数连续 | 黑桃 5-6-7 |
| 3 | 同花 (Flush) | 三张花色相同 | 红心 3-7-J |
| 4 | 顺子 (Straight) | 三张点数连续 | 5-6-7 |
| 5 | 对子 (Pair) | 两张点数相同 | 3-3-7 |
| 6 | 散牌 (High Card) | 不构成以上任何牌型 | 2-7-J |

**特殊规则**:
- A-2-3 是最小的顺子
- Q-K-A 是最大的顺子
- 同牌型比较时，先比最大牌点数，依次往下
- 点数大小: A > K > Q > J > 10 > 9 > 8 > 7 > 6 > 5 > 4 > 3 > 2

### 3.3 游戏流程

```
开始新局
  ├── 所有玩家放入底注 (ante)
  ├── 每人发 3 张暗牌
  ├── 从庄家下一位开始，顺时针轮流行动
  │   ├── 暗注状态（未看牌）:
  │   │   ├── 跟注 (Call): 放入当前注额
  │   │   ├── 加注 (Raise): 放入当前注额的 2 倍（成为新的当前注额）
  │   │   ├── 看牌 (Check): 查看自己的牌，进入明注状态
  │   │   └── 弃牌 (Fold): 放弃本局
  │   └── 明注状态（已看牌）:
  │       ├── 跟注 (Call): 放入当前注额的 2 倍
  │       ├── 加注 (Raise): 放入当前注额的 4 倍（成为新的当前注额）
  │       ├── 比牌 (Compare): 选择一个对手进行比牌（需支付比牌费用）
  │       └── 弃牌 (Fold): 放弃本局
  ├── 继续轮转直到:
  │   ├── 仅剩一人未弃牌 → 该玩家获胜
  │   ├── 有人发起比牌且获胜 → 获胜者赢得底池
  │   └── 达到最大轮数 → 剩余玩家强制比牌
  └── 结算，胜者获得底池全部筹码
```

### 3.4 比牌规则

- 只有已看牌的玩家才能发起比牌
- 比牌需要支付额外费用（等于当前跟注费用）
- 比牌的对手可以是任意未弃牌的玩家
- 牌小的一方直接出局（视为弃牌）
- 牌型完全相同时，发起比牌方输

### 3.5 筹码与下注

- 每位玩家初始筹码: 可配置（默认 1000）
- 底注 (Ante): 可配置（默认 10）
- 单局下注上限: 可配置（默认 200）
- 暗注玩家下注额 = 当前注额 × 1 (跟注) 或 × 2 (加注)
- 明注玩家下注额 = 当前注额 × 2 (跟注) 或 × 4 (加注)

## 4. 功能需求

### 4.1 游戏大厅 (P0)

- **开始新游戏**
  - 选择 AI 对手数量 (1-5)
  - 为每个 AI 对手选择驱动模型 (OpenAI / Anthropic / Gemini 等)
  - 配置初始筹码和底注
  - 每个 AI 对手显示名字和头像（预设或随机生成）
- **继续游戏**: 恢复之前未结束的游戏（可选，P1）

### 4.2 游戏主界面 (P0)

- **牌桌布局**
  - 中央显示底池筹码
  - 环形排列所有玩家位置
  - 每个玩家位置显示: 头像、名字、筹码数、当前状态（暗注/明注/已弃牌）
  - 拟物风格的扑克牌渲染
- **玩家操作区**
  - 根据当前状态显示可用操作按钮
  - 加注时可选择加注倍数
  - 比牌时可选择比牌对手
- **游戏信息**
  - 当前轮次显示
  - 行动日志（谁做了什么操作）
  - 当前注额提示

### 4.3 AI 决策系统 (P0)

- **独立 Agent**
  - 每个 AI 对手是独立的 Agent 实例
  - 通过指定的 AI Model API 进行决策
  - 拥有预设的性格特征（影响 system prompt）
- **决策输入** (AI 每次决策时获得的信息)
  - 自己的手牌（如果已看牌）
  - 当前底池大小
  - 自己的剩余筹码
  - 各对手的筹码和状态
  - 本局历史行动记录
  - **本局聊天记录**（牌桌上所有人说过的话）
  - **经验回顾摘要**（如果触发了经验回顾，见 4.8）
  - 历史对局统计（可选上下文）
- **决策输出**
  - 具体行动 (fold / call / raise / check_cards / compare)
  - 如果是 compare，指定对手
  - **可选的牌桌发言**（说一句话，或者选择沉默）
  - 心路历程记录（结构化 + 叙事）

### 4.4 心路历程系统 (P0)

#### 4.4.1 每步决策记录（结构化）

```json
{
  "round": 3,
  "turn": 2,
  "hand_evaluation": "我有一对 K，牌力中上",
  "opponent_analysis": "玩家A连续加注，可能有大牌；玩家B一直暗注跟注，难以判断",
  "risk_assessment": "中等风险，底池已经较大",
  "decision": "call",
  "reasoning": "一对K牌力不错，但对手加注凶猛，选择跟注观察",
  "confidence": 0.65,
  "emotion": "谨慎"
}
```

#### 4.4.2 叙事式总结（每局结束后生成）

以第一人称视角，将整局游戏的思考过程串成一段生动的叙事。

示例:
> "这局一开始拿到一对 K，心里还是挺稳的。但第二轮玩家A突然加注，让我有点紧张——
> 他是在虚张声势还是真有大牌？我决定先跟注看看情况。到了第三轮……"

#### 4.4.3 游戏结束总结

整场游戏结束后，每个 AI 生成一份完整的回顾报告:
- 胜负统计
- 关键决策复盘
- 对各对手的印象评价
- 自我风格总结

### 4.5 结算与报告 (P0)

- **单局结算**
  - 显示各方手牌（如果进行了比牌）
  - 显示胜者和赢得的筹码
  - 可展开查看每个 AI 的本局心路历程
- **游戏结束**
  - 触发条件: 任意玩家筹码归零 / 玩家主动点击"结束游戏"
  - 显示最终排名和筹码变化曲线
  - 每个 AI 对手的完整心路历程总结报告

### 4.6 AI 个性系统 (P1)

每个 AI 对手有预设性格，影响其决策风格:

| 性格类型 | 描述 | 行为倾向 |
|----------|------|----------|
| 激进型 | 喜欢加注和虚张声势 | 频繁加注，很少弃牌 |
| 保守型 | 谨慎行事，只有好牌才跟 | 经常弃牌，很少加注 |
| 分析型 | 注重概率和数据 | 根据期望值决策 |
| 直觉型 | 靠"感觉"打牌 | 行为不太可预测 |
| 诈唬型 | 擅长虚张声势 | 弱牌也可能大幅加注 |

### 4.7 牌桌聊天系统 (P0)

牌桌上的"嘴炮"是炸金花的灵魂之一。AI 可以通过说话来施压、虚张声势、刺探对手、或者回应别人的挑衅。

#### 4.7.1 聊天参与者

- **AI 对手**: 可以主动发言，也可以回应别人的话
- **人类玩家**: 可以在聊天窗口打字发消息，AI 能看到并可能回应

#### 4.7.2 发言时机

AI 有两种发言时机:

| 时机 | 说明 | 示例 |
|------|------|------|
| **行动附带发言** | AI 做出操作（跟注/加注/弃牌等）时，可以同时说一句话 | 加注时说 "我觉得你在虚张声势" |
| **旁观插嘴** | 非自己回合时，AI 看到别人的操作或发言，有概率主动回应 | 看到对手加注时说 "哦？这么有信心？" |

**旁观插嘴的触发规则**:
- 当某玩家做出关键操作（大幅加注、突然弃牌）时，其他 AI 有一定概率被"触发"回应
- 当有人在聊天中 @某个 AI 或明显对某 AI 说话时，该 AI 大概率会回应
- 当人类玩家发送消息时，至少一个 AI 会回应
- 触发概率受 AI 性格影响（激进型更爱说话，保守型比较沉默）
- 为避免刷屏，同一事件最多触发 2 个 AI 回应

#### 4.7.3 发言内容类型

AI 的发言风格受性格影响:

| 发言类型 | 说明 | 示例 |
|----------|------|------|
| 施压 | 试图让对手弃牌 | "你确定要跟吗？代价可不小啊" |
| 虚张声势 | 故意表现自信（牌好牌差都可能） | "今天手感不错，你们小心点" |
| 试探 | 试图从对手反应中获取信息 | "你犹豫了这么久，是不是牌不太好？" |
| 嘲讽 | 心理战，动摇对手心态 | "又弃牌？今天运气不行吧" |
| 社交 | 友好闲聊，缓解气氛 | "打了这么多局了，休息一下？" |
| 回应 | 对别人的话做出反应 | "哈哈，你说的对，但我就是不弃牌" |

#### 4.7.4 聊天对决策的影响

- AI 在做决策时，会将近期聊天记录作为输入的一部分
- AI 可以从对手的言语中推断信息（但也可能被误导）
- 人类玩家的发言也会被 AI 分析和参考
- 聊天内容会被记录在心路历程中（"他说他很自信，但我觉得他在虚张声势……"）

#### 4.7.5 聊天界面

- 牌桌右侧或下方有一个聊天窗口
- 显示所有玩家的发言（带头像和名字）
- AI 发言和操作同时显示时，操作先显示，发言紧跟其后
- 人类玩家有输入框，可以随时发消息
- 聊天消息有时间戳，旧消息可滚动查看

### 4.8 AI 经验学习系统 (P0)

AI 不仅在单局内做决策，还能回顾之前几局的心路历程，从中吸取经验教训，动态调整策略。这让 AI 越打越"聪明"，也让每场游戏更有叙事感。

#### 4.8.1 经验回顾触发条件

经验回顾不是每局都做，而是在特定条件下触发:

| 触发条件 | 说明 |
|----------|------|
| **连续输牌** | 连续输掉 2 局或以上时触发 |
| **大额损失** | 单局损失超过初始筹码的 20% 时触发 |
| **定期回顾** | 每隔 5 局自动回顾一次 |
| **筹码危机** | 筹码降至初始值的 30% 以下时触发 |
| **对手行为突变** | 检测到某对手行为模式明显改变时触发 |

#### 4.8.2 回顾内容

当经验回顾被触发时，系统将最近 N 局的心路历程提供给 AI：

- **输入**: 最近 3-5 局的 ThoughtRecord + RoundNarrative
- **AI 分析**:
  - 自己的决策模式是否有问题？（是不是太激进/太保守？）
  - 哪些决策是正确的，哪些是失误？
  - 各对手的行为模式有什么规律？（谁喜欢诈唬？谁拿好牌才跟？）
  - 接下来的策略应该如何调整？

#### 4.8.3 经验回顾输出

```json
{
  "review_trigger": "consecutive_losses",
  "rounds_reviewed": [5, 6, 7],
  "self_analysis": "最近三局我太过保守，每次拿到中等牌力就弃牌，导致白白浪费了底注",
  "opponent_patterns": {
    "player_1": "这个玩家经常虚张声势，连续两次大幅加注但最后都是散牌",
    "player_2": "非常谨慎，只有好牌才会跟注到最后"
  },
  "strategy_adjustment": "接下来我要更积极一些，中等牌力也敢跟注，同时要多关注player_1的诈唬行为",
  "confidence_shift": 0.15
}
```

#### 4.8.4 经验如何影响后续决策

- 经验回顾的结论会作为**额外的 system context** 注入到后续决策的 prompt 中
- 策略调整是渐进的，不会一次性剧变（通过控制 prompt 的权重措辞）
- 回顾内容也会被记录到心路历程中，玩家可以看到 AI "恍然大悟"的过程
- 在 AI 的叙事报告中，经验回顾是重要的转折点（"第 7 局之后我意识到……"）

#### 4.8.5 用户可见性

- 经验回顾的触发和结论**不会**实时显示给玩家（避免泄露 AI 策略）
- 但在局结束后的心路历程中，玩家可以看到 AI 进行了经验回顾
- 在游戏结束的总结报告中，AI 会提及关键的策略转折点

## 5. 用户流程

### 5.1 完整游戏流程

```
进入首页
  → 点击"开始新游戏"
  → 配置对手数量和 AI 模型
  → 配置筹码和底注
  → 进入牌桌
  → [循环] 每局:
  │   → 自动发牌和收底注
  │   → 轮流决策（AI自动，玩家手动）
  │   → 局结束 → 显示结算
  │   → 可选: 查看 AI 心路历程
  │   → 自动开始下一局
  → 游戏结束（筹码归零 / 主动结束）
  → 查看最终报告和 AI 心路历程总结
```

### 5.2 单局内玩家操作流程

```
轮到玩家行动
  → 查看界面提示（当前状态、可用操作）
  → (可选) 在聊天窗口发送消息
  → 选择操作:
  │   ├── 暗注状态: 跟注 / 加注 / 看牌 / 弃牌
  │   └── 明注状态: 跟注 / 加注 / 比牌(选对手) / 弃牌
  → 确认操作
  → 等待其他玩家（AI）行动
  → 观看 AI 行动 + AI 可能的发言
  → (可选) 随时在聊天窗口发消息，AI 可能回应
```

## 6. 非功能需求

| 项目 | 要求 |
|------|------|
| 响应时间 | AI 决策响应 < 10s（取决于模型 API） |
| 并发 | 单用户使用，无需考虑多用户并发 |
| 数据持久化 | 游戏状态持久化到本地（支持刷新恢复） |
| 浏览器兼容 | 支持 Chrome / Safari / Firefox 现代版本 |
| 部署 | 本地开发环境运行即可（docker-compose 可选） |
| 安全 | AI API Key 仅在后端使用，不暴露给前端 |
